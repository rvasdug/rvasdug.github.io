### Description
Have you ever changed one piece of code and something seemingly unrelated breaks? Or fixed a bug and 3 more pop up? Have you ever had to implement an interface but the interface is so massive that it would take all day? How about having to swap out an external service for another one, but its so hardcoded you might as well rewrite everything? If any of these scenarios sound familiar then this talk is for you. If you follow the SOLID principles you will avoid the pitfalls that cause these unfortunate programming frustrations. The goal of this talk is to use explanations, examples, and discussion so that everyone knows how to apply the SOLID principles to their code.

### Speaker: Daniel J. Dixon
From programming industrial equipment for rock quarries to programming inventory management software at warehouses Daniel has been producing various kinds of code professionally for 6 years. He's used many languages most notably: C#, Java, Angular, Python, NodeJS, and Golang. He now works as a software engineering consultant at Terazo.
